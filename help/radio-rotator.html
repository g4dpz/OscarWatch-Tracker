Before you connect

  • Use a USB serial cable or proper interface to the radio and rotator.
  • Radio and rotator need different COM ports (or device paths on Linux/macOS).
  • Close other programs that might hold the port (HRD, WSJT-X rig control, etc.).
  • For normal cross-band passes, put the radio in satellite mode (or the mode your rig manual describes) if OscarWatch has not already done so when tracking starts.
  • For receive-only modes (uplink 0 in the database — ISS SSTV, beacons, telemetry), you do not need satellite mode: OscarWatch turns it off and tunes the downlink only.

What CAT does during a pass

  • Sets satellite/split layout and modes at pass start (rig-dependent), including Voice/CW from the frequency panel.
  • Updates RX and TX frequencies for Doppler as the satellite moves (receive-only when uplink is 0).
  • Applies CTCSS from the frequency panel when supported (cross-band uplink only).
  • Respects your RX offset from the frequency panel.

Receive-only / beacon modes (uplink 0)

Examples: ISS SSTV (VHF), ISS SSTV (UHF), CW beacons. The frequency panel shows uplink as a dash. At pass start OscarWatch:

  • ICOM IC-910 / IC-9100 / IC-9700 — turns satellite mode off, clears CTCSS on Main and Sub (leftover from a previous FM satellite pass), swaps Main/Sub if needed so the downlink band is on Main, sets downlink mode and frequency on Main, and applies Doppler on Main only.
  • Other supported rigs — leaves satellite layout according to that driver (often split/VFO A+B); only the receive frequency is updated.

You do not need to select CTCSS for these modes unless you add a tone in the database and panel deliberately. After a cross-band pass, switching to ISS SSTV is enough — OscarWatch reconfigures the radio at the next track start.

Pause CAT in Settings stops frequency commands while you can still see computed frequencies on screen.

Doppler CAT thresholds

Under Settings → Radio, Doppler threshold FM and Doppler threshold SSB/CW control how often OscarWatch sends new uplink and downlink frequencies to the radio during a pass. They work like SatPC32: the rig is updated only when the calculated frequency has moved by more than the threshold since the last CAT write. Smaller values mean more frequent updates (and more serial traffic); larger values mean fewer writes. On steep or overhead passes, Doppler changes fastest near TCA (time of closest approach); a lower SSB/CW threshold or Pause CAT while you tune by hand may help if tracking feels off.

OscarWatch picks the threshold from the downlink mode in the transponder database (the mode you selected in the frequency panel):

Downlink modeSetting usedTypical use
FM / FMNDoppler threshold FMFM cubesats, ISS cross-band FM
USB / LSB / CWDoppler threshold SSB/CWLinear voice and CW on transponders

Ballpark numbers

  • FM (default 350 Hz) — On a ~5 kHz FM channel, Doppler drift is small compared with the passband. You do not need SSB-style precision. Defaults around 200–500 Hz are usually fine: the radio stays close enough without constant VFO churn. Try 150–250 Hz if you want slightly tighter tracking; go higher if your rig or USB link struggles with frequent CAT.
  • SSB/CW (default 50 Hz) — On a linear transponder you often listen in a narrow slice of the passband, so smaller steps matter. Many operators use 20–100 Hz. 50 Hz is a reasonable default. Set 0 Hz to push every correction (maximum CAT activity — only if you need it).

These thresholds apply to automatic Doppler writes. On linear USB, LSB, and CW modes, OscarWatch also watches the Main (receive) dial when you tune by hand: movements of more than 1 Hz from the last CAT receive frequency are treated as knob tuning (passband adjust); smaller differences are treated as Doppler lag. While the dial is moving, automatic CAT is paused. After you stop turning, OscarWatch waits for eight stable dial reads (~800 ms) before resuming automatic RX/TX Doppler writes.

On rigs that use Main/Sub (ICOM IC-910 / IC-9700 / IC-9100, Kenwood TS-2000 in SATL), receive updates on Main as soon as the dial is stable. Uplink (Sub) CAT is deferred for about 2.5 seconds after the last dial movement so brief pauses while scanning the passband do not hop the radio to Sub or fight your tuning. OscarWatch then returns the selected VFO to Main for receive. FM modes do not use this interactive path — both legs update automatically from the Doppler thresholds.

CAT delay (ms) on the same settings page is separate: it spaces out CI-V (and similar) commands to protect the serial link. It does not replace the Doppler thresholds above.

Rotators

GS-232Yaesu G-5500 and many clones

Configure under Settings → Rotator:

  • COM port and rotator type.
  • Azimuth range 360° or 450° (G-5500 style).
  • Smart azimuth (450°) — shortest path across north when useful.
  • Track-start elevation — do not track below this angle (e.g. 0° or a few degrees).
  • Park positions when you press Park in the sidebar.
  • Calibration offsets — optional azimuth and elevation corrections (degrees) added to pass tracking and manual rotate. Park uses your configured park az/el exactly (the sidebar shows that park position, not offset-adjusted values). Use offsets if the beam consistently stops short of or overshoots the target during tracking.

Standby

Standby parks the rotator and pauses CAT. Use it when browsing passes without transmitting or moving the beam. Press Resume when you are ready to operate — CAT and pass tracking resume for the focused satellite.

Manual rotator (Standby only)

While in Standby, the menu bar shows Rotator (only if rotator control is enabled in Settings). Open it to point the beam manually without resuming pass tracking — useful for a quick contact on a terrestrial repeater or a satellite between scheduled passes.

  • Enter Azimuth and Elevation (degrees), within your configured az/el limits.
  • Rotate — slews to the entered position.
  • Stop — stops a slew in progress (GS-232 S command; EasyComm has no standard stop).
  • Park — returns to your configured park position from Settings → Rotator.
  • Close — dismisses the dialog; the beam stays where you left it until you Park or Resume.

The dialog shows the current polled rotator position when connected. Press Resume in the sidebar when you want OscarWatch to track passes again.

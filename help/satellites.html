Select satellites

Satellites → Select satellites opens a list of spacecraft from the TLE catalogue. Tick only the ones you want on the map and in the pass list. Too many satellites can make the map busy.

TLEs (orbital data)

Pass times and positions come from Two-Line Elements downloaded from tle.oscarwatch.org. OscarWatch caches them on your PC.

  • Satellites → Refresh TLEs — update now.
  • Settings → Tracking — choose when to refresh automatically (e.g. on startup, every few hours).
  • The status bar shows how long ago TLEs were updated (e.g. “TLE 2 h ago”).
Important: Stale TLEs mean pass times and azimuth/el can be wrong. Refresh before a serious operating session.

Transponder database

Frequencies and modes are stored separately from TLEs, in your user database:

%AppData%\OscarWatch\satellite_database.json

Satellites → Manage transponder database lets you add satellites (pick a name from your TLE catalog or type a custom name), edit uplink/downlink frequencies in kHz, set mode names, Doppler type (normal or reverse), and CTCSS tones. Changes are saved to your AppData copy, not the program installation folder.

OscarWatch publishes an updated transponder list at tle.oscarwatch.org/satellite_database.json. Use Satellites → Update transponder database… (or Check for updates in the editor) to download and merge new satellites and modes. In the editor, Import JSON… and Export JSON… use a file picker for backups or sharing; import opens the same merge dialog as remote updates (press Save in the editor to keep changes).

  • Settings → Tracking — enable Check for transponder database updates on startup (on by default).
  • New remote entries are selected by default in the merge dialog; uncheck any you do not want.
  • If a mode you already have differs from the published copy, your version is kept unless you choose Use published version.
  • Restore defaults reloads the database shipped with your app version, not the remote file.
  • Set uplink to 0 for receive-only modes (SSTV, beacons, telemetry).

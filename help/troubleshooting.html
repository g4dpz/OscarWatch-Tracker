macOS — app or recording blocked on first install

Release builds from GitHub are not code-signed or notarized. macOS may block OscarWatch or bundled native libraries the first time you run them. These steps are usually one-time per install.

  • OscarWatch — if Finder says the app “cannot be opened”, right-click OscarWatch → Open once, or use System Settings → Privacy & Security → Open Anyway.
  • Pass recording — if you use automatic WAV capture, macOS may also ask you to allow libportaudio.dylib (in runtimes/osx-*/native/ inside the app folder). Approve it the same way when prompted.
  • Microphone — allow microphone access when you first enable recording in Settings → Recording.

Tracking, passes, and radio/rotator control work without pass recording if you skip the PortAudio step.

Pass times look wrong

  • Refresh TLEs (Satellites → Refresh TLEs).
  • Check station latitude and longitude in Settings.
  • Confirm your PC clock and time zone are correct (UTC is shown in the status bar).

No frequencies / wrong mode

  • Select the satellite on the map or pass list.
  • Pick a transponder mode in the frequency panel.
  • Edit or add the satellite in Manage transponder database.

Rotator or radio not connecting

  • Check COM port in Settings — use Device Manager on Windows to see which port appears when you plug in USB.
  • Ensure radio and rotator use different ports.
  • Close other programs using the same port.
  • Leave Standby if you expect tracking — Resume for live operation.
  • Sidebar may show a COM port conflict warning if both devices share one port.

CAT frequencies not updating

  • Check Pause CAT is off in Settings → Radio.
  • Confirm you are not in Standby mode.
  • Verify rig type and CI-V address (ICOM).
  • For cross-band modes, the radio should be in satellite/split layout (OscarWatch usually sets this at pass start).
  • For receive-only modes (e.g. ISS SSTV with uplink 0), satellite mode should be off — OscarWatch does that on ICOM rigs. Pick the matching band mode (SSTV VHF vs UHF) in the frequency panel.
  • On linear SSB/CW, check Doppler thresholds — very large values can make tracking look sluggish near TCA.

Dual FT-817 / FT-818 not connecting

  • Enable Settings → Radio → Dual radio and set a COM port for both downlink and uplink.
  • FT-817/818 appear only in the dual-radio lists — not as a single-rig type.
  • Match menu #14 baud on each radio to Settings on that leg (suggested 4800; 38400 also works).
  • Close Hamlib or other CAT programs using either COM port before starting OscarWatch.
  • After switching from another rig (e.g. IC-910), quit OscarWatch, replug USB, and restart if CI-V or CAT stops responding.

FT-817 / FT-818: downlink stuck, uplink tracks

  • Use Full doppler in the frequency panel (not TX-only or RX-only).
  • On linear SSB, spin the downlink to find the signal, then pause ~1 s so passband trim is captured before doppler resumes.
  • On FM, the dial is locked via CAT during tracking — use Pause CAT in Settings if you need to tune manually.

ICOM: radio stays on Sub or uplink lags after tuning

On IC-910 / IC-9700 / IC-9100 in satellite mode, OscarWatch keeps you on Main for receive while you turn the dial on linear passes. Uplink (Sub) CAT is intentionally delayed for about 2.5 seconds after you stop moving Main so scanning the passband does not select Sub. If uplink seems late right after a long tune, wait a moment — it should catch up. If Sub stays selected with no pass running, press Resume or toggle Standby, or turn satellite mode off and on on the radio. See passband knob tuning in Radio & rotator help.

Voice announcements silent

  • Enable voice in Settings → Voice and set trigger elevation.
  • On Linux, install espeak-ng or speech-dispatcher.
  • Use Test in the Voice tab.

Pass recording not working

  • Enable recording in Settings → Recording and press Save.
  • Select an input device — your radio line-in or USB audio interface, not the default microphone unless that is what you use.
  • Press Refresh on the device list if you plugged in hardware after opening Settings.
  • Use Test recording (5 s) to confirm audio levels before a pass.
  • Recording only runs for the focused satellite — click it on the map or pass list first.
  • Check start/stop elevations: the satellite must rise through the start elevation while selected.
  • On Linux, ensure PulseAudio or ALSA is running (libasound2 on Debian/Ubuntu).
  • On macOS, see macOS — app or recording blocked if libportaudio.dylib is blocked; grant microphone permission when prompted.
  • Recordings save to %AppData%\OscarWatch\recordings\ unless you chose another folder.

Logs

Help → Open logs folder opens %AppData%\OscarWatch\logs\. Log files help diagnose CAT and rotator errors. API keys are not written to logs.

Check the project README and issue tracker on GitHub. Include your OscarWatch version (Help → About), operating system, radio/rotator models, and a relevant log excerpt.

Open from the menu: Settings. Press Save when finished. Settings are stored in %AppData%\OscarWatch\settings.json.

Station

Your operating position: name (for labels), latitude, longitude, Maidenhead grid, and height above sea level in metres. The grid square is normalised to uppercase as you type. Accurate height helps slightly with geometry; position is essential for passes and az/el.

TLE

Choose where pass times and orbital positions come from. The transponder frequency database is separate — see Transponder database.

  • OscarWatch (tle.oscarwatch.org) — default AMSAT-oriented catalogue (recommended).
  • AMSAT.org (nasabare.txt) — the published set from amsat.org/tle/current/nasabare.txt.
  • Custom URL — any HTTP(S) address that returns a standard two-line TLE text file (for example Celestrak amateur-group TLEs).
  • Local file — browse to a .txt or .tle file on your PC; use Browse… to pick it.
  • TLE updates — when to download fresh data automatically (OscarWatch and custom URLs only). Local files are re-read when you save Settings or use Satellites → Refresh TLEs.

Changing the source and pressing Save reloads the catalogue. The status bar shows which source is active (for example “TLE 2 h ago (tle.oscarwatch.org)”).

Tracking

  • Minimum elevation — passes below this angle are hidden from predictions.
  • Prediction hours — how far ahead to compute passes.
  • Check for transponder database updates on startup — downloads tle.oscarwatch.org/satellite_database.json when the app opens and offers to merge new satellites/modes (on by default). Use Satellites → Update transponder database… to check manually.

Appearance

Light, dark, or follow Windows theme. The sky plot follows the theme; the world map image stays light for readability.

Language — English (UK) is the default UI language (British English spelling and en-GB date/time formatting). Japanese, Portuguese (Brazil), and Simplified Chinese are also available. Restart is not required; the choice applies when you press Save.

Show footprint motion arrows — optional arrows inside satellite footprints on the world map indicating ground-track direction (on by default).

Voice

Optional spoken alert when a satellite rises above a set elevation (e.g. “Alpha Oscar Zero Seven is rising”). Choose a voice and test it. On Linux you may need espeak-ng installed.

Recording

Optional automatic WAV capture from a line-in or USB audio device while you work a pass. Open Settings → Recording to configure it.

  • Enable automatic pass recording — turn recording on or off.
  • Input device — choose the audio input that carries your radio audio (sound card line-in, USB interface, etc.). Press Refresh if you plug in hardware after opening Settings.
  • Format — mono 44.1 kHz is a good default for FM/SSB; stereo if your interface requires it.
  • Start / stop elevation — recording begins when the focused satellite rises through the start elevation, and stops when it falls below the stop elevation. Keep stop at or below start to avoid flapping near the threshold.
  • Output folder — leave blank to use the default %AppData%\OscarWatch\recordings\.
  • Test recording (5 s) — saves a short clip so you can confirm levels and device selection before a pass.

Files are named in UTC: {satellite-name}-{yy-MM-dd-HH-mm}.wav (for example so-50-26-05-24-14-30.wav). Only the satellite you have selected on the map or pass list is recorded. If you switch to another satellite that is already above the start elevation, recording begins straight away. While recording, a red REC badge appears on that pass in the sidebar.

WAV files are uncompressed (~5 MB per minute mono at 44.1 kHz). Convert to MP3 elsewhere if you need smaller files.

Rotator & Radio

See Radio & rotator for COM ports, rig type, Doppler thresholds (FM vs SSB/CW ballpark values), and rotator options.

Under Radio, Linear CW: keep receive in USB/LSB (transmit CW only) controls how CAT and Cloudlog set modes when the frequency panel is on CW for linear SSB satellites (see Frequencies).

Integrations

The Integrations tab groups optional hardware and external services: GPS, hams.at Upcoming Roves, and Cloudlog. Pick a section from the list on the left.

GPS

Choose Serial (COM port) for a USB or serial GPS on its own port (not shared with the radio or rotator), or gpsd (network) when a gpsd daemon on your LAN (for example on a Raspberry Pi) already manages the receiver.

  • Enable GPS — turn GPS integration on or off.
  • Connection — Serial reads NMEA GGA / RMC sentences; gpsd uses the JSON watch stream on TCP port 2947 (host + port configurable).
  • Serial port / baud rate — when using serial, match the GPS (often 4800 or 9600 8N1). Press Refresh if the port list is stale.
  • gpsd host / port — hostname or IP of the machine running gpsd (often 127.0.0.1 on the same PC, or your Pi’s LAN address).
  • Refresh status — shows connected / waiting for fix / fix with lat/lon and satellite count.
  • Apply fix to station — copies the current fix into Settings → Station (latitude, longitude, grid, and optionally altitude).
  • Automatically update station position from GPS fix — keeps your QTH in sync while roving (saves periodically).
  • Update altitude from GGA — when auto-update is on, applies height from the GGA sentence.
  • Use GPS UTC for satellite tracking — pass predictions and live tracking use GPS time instead of the PC clock. Does not change the Windows system clock.
  • Minimum satellites for fix — ignore fixes with fewer satellites in view (default 3).

With GPS enabled, the status bar shows GPS in green when a fix is active and red when GPS is on but there is no fix. If Use GPS UTC is checked, GPS Time appears the same way (green when tracking is using GPS UTC).

hams.at

Enable the integration, paste your hams.at API key (Bearer token), and set how often roves refresh. Your key is stored locally in settings.json like other credentials. Use Test connection to verify the key. Set your observer location on hams.at so is_workable alerts match your station.

Cloudlog

If you use Cloudlog, enter your server URL and API key. Click Test connection after both fields are filled — OscarWatch loads your available logbooks (logbooks must have a public slug in Cloudlog) and shows a Logbook dropdown. Pick the logbook to use for grid lookups on hams.at roves.

While tracking, OscarWatch posts current satellite uplink/downlink frequencies to the Radio API when they change, and repeats the same state on a keepalive interval (default 10 minutes). Configure Radio name and keepalive below the logbook. Use Test connection before relying on live updates in the field.

Highlight grids I still need on hams.at roves (shown when a logbook is selected) checks each rove grid against that logbook using Cloudlog’s logbook_check_grid API with band: SAT, so only satellite QSOs count. Grid badges on rove rows are described in Map & sidebar → Rove grid badges.
